# CopilotUsageFilter

<p align="center">
  <img src="assets/128.png" alt="CopilotUsageFilter" width="96"/>
</p>

A minimal Windows system-tray utility that intercepts GitHub Copilot CLI's OpenTelemetry traces, prints per-interaction token usage to the console, and back-patches the counts into Copilot's session-state files.

---

## What it does

When the Copilot CLI completes a chat turn it exports an OTLP span to `http://localhost:4318/v1/traces`. CopilotUsageFilter:

1. Receives the span over HTTP.
2. Extracts `input_tokens`, `output_tokens`, `cache_creation_tokens`, `cache_read_tokens`.
3. Prints a colored, tab-separated line to the console:

```
2026-05-22T23:16:05    session=450d63d8    interaction=5f961441    turn=1    input=300    output=77    cache_write=20    cache_read=8
```

4. Patches the matching `assistant.message` event in  
   `%USERPROFILE%\.copilot\session-state\<session-guid>\events.jsonl`  
   with the four token fields (only if not already present).

## Usage

### 1. Start the collector

From any terminal (cmd, PowerShell, or Windows Terminal):

```powershell
.\CopilotUsageFilter.exe          # default port 4318
.\CopilotUsageFilter.exe 4319     # custom port
```

The program attaches to your terminal's console automatically. The first output line confirms the listener is ready:

```
2026-05-22T23:16:04    Listening    http://localhost:4318
```

A tray icon appears in the system notification area.

### 2. Tell the Copilot CLI where to export

Set the OTLP endpoint environment variable **before** running `copilot`:

```powershell
$env:OTEL_EXPORTER_OTLP_ENDPOINT = "http://localhost:4318"
copilot chat "explain this code"
```

Or set it permanently in your user environment:

```powershell
[Environment]::SetEnvironmentVariable(
  "OTEL_EXPORTER_OTLP_ENDPOINT",
  "http://localhost:4318",
  "User"
)
```

### 3. Read the output

Each completed chat turn produces one log line with tab-separated fields:

| Field | Description |
|---|---|
| `<timestamp>` | ISO-8601 local time (`yyyy-MM-ddTHH:mm:ss`) |
| `session=` | First 8 chars of the Copilot session GUID |
| `interaction=` | First 8 chars of the interaction GUID |
| `turn=` | Turn index within the interaction (0, 1, …) |
| `input=` | Input tokens sent to the model |
| `output=` | Output tokens generated by the model |
| `cache_write=` | Tokens written to the prompt cache |
| `cache_read=` | Tokens read from the prompt cache |

Lines are colored per-field when output goes to a live console. Colors are suppressed automatically when you pipe or redirect output.

## Session-state patching

Copilot stores conversation history in:

```
%USERPROFILE%\.copilot\session-state\<session-guid>\events.jsonl
```

After each chat span is received, CopilotUsageFilter finds the `assistant.message` event whose `interactionId` and `turnId` match the span, then appends:

```json
"input_tokens": 300,
"output_tokens": 77,
"cache_creation_tokens": 20,
"cache_read_tokens": 8
```

The write is atomic (`.tmp` + `File.Replace`) so the file is never left in a corrupt state.

## Troubleshooting

**No output after running `copilot chat`**

- Confirm the env var is set: `$env:OTEL_EXPORTER_OTLP_ENDPOINT`
- Check the tray icon is present and shows `Listening on :4318`
- Try posting a test span manually:
  ```powershell
  Invoke-WebRequest -Uri http://localhost:4318/v1/traces -Method Post `
    -ContentType "application/json" `
    -Body '{"resourceSpans":[{"scopeSpans":[{"spans":[{"attributes":[
      {"key":"gen_ai.operation.name","value":{"stringValue":"chat"}},
      {"key":"gen_ai.usage.input_tokens","value":{"intValue":"10"}},
      {"key":"gen_ai.usage.output_tokens","value":{"intValue":"5"}}
    ]}]}]}]}'
  ```

**Port already in use**

Pass a different port as the first argument:

```powershell
.\CopilotUsageFilter.exe 4319
$env:OTEL_EXPORTER_OTLP_ENDPOINT = "http://localhost:4319"
```

**`events.jsonl` not patched**

- Verify `%USERPROFILE%\.copilot\session-state\` exists and contains session folders.
- The span must include `gen_ai.conversation.id` (session GUID) for the patcher to locate the correct file.
